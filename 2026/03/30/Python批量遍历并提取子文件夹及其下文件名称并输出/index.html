二、核心依赖库

代码用到两个 Python 常用库，需提前安装：

os：Python 内置库，用于文件 / 文件夹的路径处理、遍历，无需额外安装；

pandas：第三方数据处理库，用于将统计数据整理为表格并输出 Excel，需先安装 pandas 输出 Excel 的依赖 openpyxl

三、固定配置区

代码开头设置了可直接复用的固定配置项，用户无需改动代码逻辑，仅需保证文件目录符合要求即可：

main_folder：主文件夹名称为“20xx年以来xx文件”，是统计的根目录；

excel_name：输出的 Excel 文件名，统计结果会保存为该名称；

zip_suffix：需要识别的压缩文件后缀元组，包含.zip/.rar/.7z/.tar/.gz/.bz2，可根据实际需求增删后缀。

四、核心执行逻辑（分两大分支）

分支 1：主文件夹不存在 → 报错提示

通过os.path.exists(main_folder)检查主文件夹是否和 Python 文件在同一目录，若不存在则打印红色错误提示，告知用户目录放置要求，程序终止。

分支 2：主文件夹存在 → 开始统计（核心步骤）

初始化变量：创建空列表 result 存储统计数据，创建 index 作为 Excel 的序号，初始值为 1；

遍历一级子文件夹（项目名称）：遍历主文件夹下的所有内容，仅处理文件夹（跳过文件），该文件夹名称作为项目名称；

拼接得到该项目文件夹的完整路径 project_path；

遍历项目文件夹下的压缩文件：遍历项目文件夹内的所有内容，仅处理文件且后缀属于配置的压缩文件类型；

拼接压缩文件完整路径，通过 os.path.splitext(file_name)[0] 去掉文件后缀，将压缩文件名作为公司名称（核心提取逻辑）；

整理数据：将序号、项目名称、公司名称组成列表，添加到 result 中，序号自增 1；

生成 Excel：通过 pandas 将 result 转换为 DataFrame，设置列名为序号、项目名称、公司名称；

调用 to_excel 输出 Excel 文件，设置 index=False（不将 pandas 的行索引写入 Excel）；

执行成功提示：打印绿色完成标识，输出统计的总记录数和 Excel 保存路径。

六、代码的特点

  • 易用性：配置区和业务逻辑分离，非开发人员也能直接使用，仅需保证目录结构正确；

  • 鲁棒性：做了双层过滤（仅处理文件夹、仅处理指定压缩文件），避免无关文件干扰统计；

  • 可视化：执行结果有明确的成功 / 失败提示，包含记录数、文件保存信息，便于用户核对；

  • 可扩展：如需识别更多压缩文件类型，仅需在zip_suffix元组中添加后缀即可。

七、潜在使用注意点

核心提取逻辑为压缩文件名 = 公司名称，若压缩文件名包含额外字符（如日期、编号），需先整理文件名再运行代码；

仅遍历主文件夹的一级子文件夹，若项目文件夹内有嵌套子文件夹，嵌套的压缩文件不会被统计；

输出 Excel 时，若同名 Excel 文件已打开，会报错（需先关闭现有文件）。
