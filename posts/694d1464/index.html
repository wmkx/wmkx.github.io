Hexo+GitHub Pages 祖传关键字黑名单，杜绝标签404

发表于2026-04-08|更新于2026-05-01|基础设置
|浏览量:

搭建 Hexo + GitHub Pages 个人博客时，很多博主会遇到「玄学404」：标签首页正常、本地预览全正常，唯独部分标签线上访问报 GitHub 官方404，即便文件完整、路径匹配也无法解决。核心原因并非配置错误，而是 GitHub Pages 平台原生预留关键字拦截机制——这是 Hexo 博客圈流传多年的「祖传深坑」。本文整理完整黑名单、核心原理及最简避坑方案，帮你一次性规避。

一、核心原理：为什么会触发拦截？

GitHub Pages 有双层路由匹配规则，优先级：平台内置预留路由 > 用户静态文件路由。当访问的 URL 命中平台预留关键字时，服务器会优先走官方内部路由，完全忽略用户仓库内的同名静态文件，直接返回404。
其中，Hexo、Jekyll 作为主流建站引擎，被 GitHub 纳入核心预留黑名单；此外，平台接口、服务器相关词汇也会被拦截，这也是部分标签无故404的根本原因。
此外，GitHub Pages 服务器为 Linux 环境，路径大小写严格区分，但即便修改关键字大小写，只要词根属于黑名单，依旧会被拦截。

二、完整祖传关键字黑名单清单（必看）

以下词汇，禁止用于博客标签名、分类名、自定义页面名、文件夹名及自定义永久链接，否则必触发404拦截，按风险等级排序：

🔴 核心建站引擎黑名单（最高危，必拦截）

Hexo、Jekyll

🔴 GitHub平台原生接口黑名单

GitHub、Git、Pages、Admin、API、Help、Support、Login、Auth、User、Account

🔴 服务器&网络通用高危拦截词

DNS、IP、HTTP、HTTPS、WWW、Root、Config
注：部分词汇（如DNS）可能存在兼容例外，但不建议复用，避免后续拦截。

🔴 系统保留通用敏感词

Index、Home、Default、Error、Cache、Static

推荐零成本、易操作的方案，无需修改复杂配置，一劳永逸：

方案：标签同义改名（首选）

避开黑名单词根，修改标签名称，不影响文章归类和语义：

  • 原黑名单标签（必404）：tags: - Hexo
  • 修改后安全标签：tags: - Hexo博客 / Hexo建站 / Hexo教程
  1. 执行标准部署流程（不可跳过clean）：

四、新手避坑忠告

  • 本地正常≠线上正常：本地预览无拦截规则，新建标签先对照黑名单筛查。
  • 404优先查黑名单：文件存在、路径匹配仍404，无需死磕配置，先排查关键字。
  • 命名加后缀：建站相关标签建议加后缀（如Hexo博客），避开纯词根黑名单。
