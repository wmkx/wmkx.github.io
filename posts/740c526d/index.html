一、安装shiro主题（2种方式，推荐Git克隆）

shiro主题提供两种安装方式，Git克隆方式可方便后续主题更新，优先推荐；npm安装方式适合新手，操作更简单。

二、启用shiro主题（关键步骤）

安装完成后，需修改Hexo站点配置，启用shiro主题，步骤如下：

  1. 用VS Code打开Hexo博客根目录下的_config.yml文件（站点配置文件）。

三、安装shiro主题依赖（必做，否则样式错乱）

重点提醒：shiro主题基于Tailwind CSS开发，依赖必须在themes/shiro目录下执行，不可在博客根目录执行，否则会导致依赖安装失败、页面无样式。

四、配置shiro主题（可选，推荐自定义）

Hexo 5.0+ 支持独立主题配置文件，无需修改shiro主题源码（方便后续主题更新），推荐按以下方式配置：

  1. 在Hexo博客根目录，创建_config.shiro.yml文件（shiro主题独立配置文件）。

  2. 打开themes/shiro目录下的_config.yml文件，将其全部内容复制到_config.shiro.yml中。

  3. 按需修改配置（新手可先修改以下常用配置，比如导航配置：修改menu字段，设置博客导航栏选项，其他保持默认）。

  4. 保存配置文件，配置修改后需重新生成静态文件才能生效。

六、部署上线

七、常见问题与解决方案

问题1：shiro主题不生效，页面还是旧主题样式？

解决方案：

  • 检查站点配置文件_config.yml中theme: shiro的拼写是否正确，冒号后是否有空格，字母是否为小写。

  • 执行hexo clean清除缓存，再重新生成并预览（hexo g && hexo s）。

  • 确认shiro主题安装目录是themes/shiro，未修改文件夹名称。

问题2：页面样式错乱、空白，控制台报错“tailwindcss not found”？

解决方案：

  • 重新进入themes/shiro目录，执行npm install && npm run build，确保依赖安装完整、CSS编译成功。

  • 检查是否在博客根目录执行了npm install，导致依赖安装错误，需删除根目录下多余的依赖，重新在主题目录安装。

问题4：配置修改后不生效？

解决方案：

  • 确认配置修改的是根目录下的_config.shiro.yml文件，而非themes/shiro目录下的_config.yml（后者会被覆盖）。

  • 修改配置后，必须执行hexo clean && hexo g，重新生成静态文件才能生效。
