一、准备工作

1. 必备软件安装

搭建博客需要3个基础软件，全程默认安装即可，无需额外配置。

  • Node.js（含 npm）：下载 LTS 版本（推荐长期支持版，稳定性更高），一路点击“下一步”完成安装。

  • Git：下载 官网版本，同样默认安装，安装完成后重启电脑（确保环境变量生效）。

验证安装：打开终端，输入以下3条命令，能正常输出版本号即安装成功。

2. 账号与域名准备

  • GitHub 账号：注册并登录 GitHub（用于托管博客源码和部署页面）。

  • Cloudflare 账号：注册并登录 Cloudflare（用于DNS解析、CDN加速和抗污染）。

  • 自定义域名：在阿里云、腾讯云等平台购买一个域名（如 qingfengxiao.com），无需备案即可正常访问。

四、绑定自定义域名（Cloudflare 核心步骤）

通过自定义域名（如 qingfengxiao.com）访问博客，更简洁易记，同时借助 Cloudflare 实现抗 DNS 污染和全球加速。

1. 将域名接入 Cloudflare

（1）登录 Cloudflare(https://dash.cloudflare.com/)，推荐使用`GitHub`账号登录。点击右上角`「+ Add」→Connect a domain`（添加站点）。

（2）输入你的自定义域名（如 qingfengxiao.com），点击Continue。

（3）选择 Cloudflare 免费计划Free plan，点击「Continue」。

（4） Cloudflare 会自动扫描你的域名解析记录（无需操作），扫描完成后点击「Continue」。Cloudflare 分配 2 个 NS 服务器地址（如 dion.ns.cloudflare.com 和 teagan.ns.cloudflare.com）。

2. 配置 Cloudflare DNS 解析

NS 服务器生效后，进入 Cloudflare 域名管理页面，点击左侧「DNS → Records」，删除所有默认解析记录，添加以下2条CNAME记录：

类型（Type） 名称（Name） 内容（Content） 代理状态（Proxy status）
CNAME @ 你的用户名.github.io Proxied（橙色云）
CNAME www 你的用户名.github.io Proxied（橙色云）

3. 配置 SSL/TLS

必须配置 SSL/TLS，否则访问博客会提示“不安全”，甚至无法访问。

（1）左侧菜单 →「SSL/TLS → Overview」，将加密模式改为 Full (strict)（最安全，兼容 GitHub Pages SSL 证书）。

（2）左侧菜单 →「SSL/TLS → Edge Certificates」，开启以下两个开关：

  • Always Use HTTPS：强制所有 HTTP 请求跳转到 HTTPS，避免不安全访问。

  • Automatic HTTPS Rewrites：自动修复页面中的混合内容（HTTP 资源），避免页面显示异常。

4. GitHub 绑定自定义域名

（1）打开你的 GitHub 仓库（你的用户名.github.io），点击「Settings → Pages」。

（2）在「Custom domain」输入框中，填写你的自定义域名（如 qingfengxiao.com）。

（3）勾选「Enforce HTTPS」（强制 HTTPS 访问），点击「Save」。

（4）等待 GitHub 验证 DNS 解析，页面显示「DNS check successful」即绑定成功。
