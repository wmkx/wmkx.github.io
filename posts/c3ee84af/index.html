发表于2026-04-29|更新于2026-04-30|建站教程

一、更新前说明（必看）

  1. Twikoo 必须前后端版本一致，否则会一直提示：
    前端版本 x.x.x，云函数版本 x.x.x，请升级

  2. latest ≠ 自动更新，Vercel 不会自动升级云函数，必须手动操作触发更新

  3. 正确更新顺序（不可颠倒）：
    更新 Vercel 云函数 → 更新博客前端 CDN（可选镜像站） → 重新部署博客

  4. 镜像站说明：默认 CDN（jsdelivr）可能存在国内加载慢、偶尔无法访问的问题，推荐使用 npmmirror 国内镜像站，加载速度更快、更稳定，教程中会同步提供两种 CDN 配置供选择。

二、更新 Vercel 后端云函数（1 分钟搞定）

方法 A：最简单（推荐，不用改代码）

  1. 打开 Vercel → 进入你的 Twikoo 项目

  2. 点击顶部 Deployments（部署记录）

  3. 找到最新的一条部署记录 → 点击右侧 三个点 ⋮

  4. 选择 Redeploy（重新部署）

  5. 重点：一定要取消勾选「Use existing Build Cache」（取消缓存，才能强制拉取最新版本）

  6. 点击「Deploy」，等待部署状态变成绿色 ✅，即表示云函数更新完成。

完成后，云函数会自动更新为 Twikoo 最新稳定版，可在部署日志中查看具体版本号（如 1.7.7）。

方法 B：固定版本（更稳定，避免后续版本错乱）

  1. 进入你的 Twikoo 项目 Git 仓库（GitHub/Gitee），找到并打开 package.json 文件

  2. 修改 dependencies 中的 twikoo-vercel 版本号，写死最新稳定版（以 1.7.7 为例）：{ "dependencies": { "twikoo-vercel": "1.7.7" } }

  3. 点击「Commit changes」提交修改，Vercel 会自动触发重新部署，部署完成后云函数版本即固定为你设置的版本。

五、验证更新是否成功

  1. 打开你的博客，进入任意文章的评论区；

  2. 点击评论区的「登录」，进入 Twikoo 管理面板；

  3. 查看面板顶部的提示，若显示：前端版本 x.x.x，云函数版本 x.x.x，且两个版本号完全一致，即表示更新成功；

六、最常见 4 个坑（必看，避坑必看）

  1. 没取消 CDN 配置中 twikoo: 前面的 # 号 → 配置不生效，前端仍加载旧版本；

  2. 前后端版本号不一致（CDN、云函数、twikoo 配置中的 version 三者不统一）→ 一直提示版本不匹配；

  3. Vercel 重新部署时没取消「Use existing Build Cache」→ 云函数不更新，仍停留在旧版本；

  4. 默认 CDN 加载慢/失败 → 切换为 npmmirror 镜像站 CDN，重新部署即可解决。

八、镜像站补充说明

  • npmmirror 是阿里巴巴旗下的 npm 镜像站，国内访问速度远超 jsdelivr，适合国内博客用户；

  • 镜像站版本同步官方，只要替换版本号，就能加载对应版本的 Twikoo 前端文件；

  • 若后续更新版本，只需同步修改「CDN 地址中的版本号」和「twikoo 配置中的 version」，再重新部署即可。
